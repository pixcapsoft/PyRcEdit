# Contributing to PyRcEdit

First of all, thank you for your interest in contributing to **PyRcEdit**! 🎉 We welcome all contributions, whether it's reporting bugs, suggesting new features, improving documentation, or submitting pull requests.

## How to Contribute

### 1. Reporting Bugs
- Ensure the bug hasn't already been reported in the issue tracker.
- Describe the bug clearly. Include steps to reproduce, what you expected to happen, and what actually happened.
- Provide any relevant system information (e.g., Windows version, Python version).

### 2. Suggesting Features
- We love new ideas! If you have a suggestion, open an issue explaining your proposed feature.
- Explain *why* this feature would be useful and how it aligns with the project's goals.

### 3. Submitting Pull Requests
- **Fork** the repository and create your branch from `main`.
- **Setting up your environment**:
  - Make sure you are on Windows.
  - Make sure you have Python 3.8+ installed.
  - Clone your fork and test your changes.
- **Code Style**:
  - Try to adhere to PEP 8 standards.
  - Write clean, readable, and well-commented code.
- **Commit Messages**: Write clear, descriptive commit messages.
- **Testing**: Before submitting, ensure that your changes work properly and don't break existing functionality. Since PyRcEdit interacts directly with Win32 APIs, it's crucial to test it on compiled executables to ensure everything acts as expected.

## Getting Started
1. Fork the repo.
2. Clone your fork locally.
3. Make your changes and test them using sample Windows PEs (e.g. `.exe` or `.dll`).
4. Push your changes and submit a Pull Request!

Thank you for contributing to PyRcEdit! 🚀
